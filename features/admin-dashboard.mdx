## User management вЂ” `/admin/usuarios`

`UserManagement.jsx` lists every user in the system.

### Filters

- **Search** вЂ” filters by name or email (case-insensitive substring match).
- **Role filter** вЂ” dropdown to show all roles or filter by `admin`, `seller`, or `client`.

### User table

Columns: name/email, role badge, active status, registration date, last login, and actions.

**Role badge colors:**

| Role | Color |
|---|---|
| `admin` | Purple |
| `seller` | Blue |
| `client` | Green |

### Actions

**Change role** вЂ” clicking the edit icon opens an inline `<select>` with options for Administrador, Vendedor, and Cliente. Saving calls `updateUserRole(userId, newRole)` from `MetricsContext` and shows a notification.

**Toggle activation** вЂ” clicking the toggle icon calls `toggleUserActive(userId)`. Active users show a red "Desactivar" toggle; inactive users show a green "Activar" toggle.

### Role summary cards

Three stat cards below the table show the total count for each role (admins, sellers, clients).

## Product catalog вЂ” `/admin/productos`

`AdminProducts.jsx` gives the admin a full view of all products across all sellers.

### Stock metrics

Four cards at the top show total products, in-stock count, low-stock count (< 20 m), and out-of-stock count, derived from `calculateStockMetrics(products)`.

### Filters

- Text search by product name or category.
- Category dropdown populated from unique category values in the product list.

### Product table

Columns: product image + name, category, price (COP), stock (m), and actions.

**Inline editing** is triggered by the edit button. Price and stock become number inputs. Save icon (`FiDollarSign`) commits via `updateProduct(productId, { price, stock })`. The delete button (`FiTrash2`) calls `deleteProduct(productId)` after a browser confirmation.

Stock values below 20 are shown in red; 20 and above are shown in green.

## Carousel management вЂ” `/admin/carrusel`

`ManageCarousel.jsx` controls the homepage promotional carousel. The carousel slides are stored in `UIContext` via the `carouselSlides` array.

**Adding a slide:**

<Steps>
  <Step title="Open the form">
    Click "Agregar Slide" to expand the new-slide form.
  </Step>
  <Step title="Fill in the fields">
    Enter a title (required), subtitle (optional), and upload an image (required). Images are read as Base64 data URLs.
  </Step>
  <Step title="Save">
    Click "Guardar Nuevo Slide" to call `addSlide(newSlide)` in `UIContext`.
  </Step>
</Steps>

**Editing a slide:** Click the edit icon on any slide row. Title, subtitle, and image can all be changed. Save with the `FiSave` icon or cancel with `FiX`.

**Deleting a slide:** Click the trash icon (`FiTrash2`) вЂ” a confirmation dialog is shown before `removeSlide(id)` is called.

## Homepage section management вЂ” `/admin/home`

`ManageHome.jsx` controls which sections are visible on the homepage.

Section visibility is persisted by posting to `POST http://localhost:8081/api/config` with `key: 'home_sections_config'` and the updated sections JSON as the value. On mount, the page fetches the current config from `GET http://localhost:8081/api/config/home_sections_config`.

**Manageable sections:**

| Section ID | Name | Description |
|---|---|---|
| `hero` | Hero Section | Main banner and CTA |
| `carousel` | Carrusel de Promociones | Featured slides |
| `featured` | Productos Destacados | Best-products grid |
| `benefits` | SecciГіn de Beneficios | Trust icons |

Each section has a toggle button. When visible, it shows green "Visible"; when hidden, it shows gray "Oculto".

## Bug reports вЂ” `/admin/reportes`

`AdminBugReports.jsx` lists all bug reports, feature requests, and visual issues submitted by users.

### Report table

Columns: ID, title/type/date, severity badge, status, submitting user, and actions.

**Severity badges:**

| Severity | Color |
|---|---|
| `low` | Green |
| `medium` | Blue |
| `high` | Orange |
| `critical` | Red |

**Report types:** `bug`, `visual`, `feature`.

**Actions:**

- **Resolve** (`FiCheck`) вЂ” sets status to `closed`.
- **Discard** (`FiX`) вЂ” sets status to `rejected`.

## Seller metrics вЂ” `/admin/vendedores`

`SellerMetrics.jsx` provides a cross-seller performance comparison.

A bar chart (`BarChart`) compares total sales across all sellers. The table below shows each seller with columns for name/email, total sales, order count (with completed subtotal), average ticket, quality report count (color-coded by volume), and active status.

Clicking a seller row expands an inline detail panel with four stat cards:

- Ingresos Generados
- Pedidos Completados (with pending count)
- Ticket Promedio
- Reportes Recibidos (with report rate as percentage of orders)

Three summary cards at the bottom show the best seller, the average sales per seller, and the total report count across all sellers.

## Client metrics вЂ” `/admin/clientes`

`ClientMetrics.jsx` segments clients by total spend and shows a top-10 leaderboard.

### Segmentation

Three cards show client counts and total value by segment:

| Segment | Threshold |
|---|---|
| Clientes VIP | Total spend > $1,000,000 COP |
| Clientes Regulares | $300,000 вЂ“ $1,000,000 COP |
| Clientes Nuevos | < $300,000 COP |

Segmentation is computed by `segmentClients(clients, orders)` from `metricsUtils`.

### Top 10 clients table

Columns: rank (gold/silver/bronze podium badges for positions 1вЂ“3), name/email, total spent, purchase count, average order value, purchase frequency (days between orders), and last purchase date.

### Summary cards

Four aggregate stats at the bottom: average client value, average purchases per client, best client (name + total), and average ticket.
