## Sales heat map

**Route:** `/admin/analytics/mapa-ventas`  
**Component:** `src/pages/admin/Analytics/SalesHeatMap.jsx`

The heat map visualises revenue distribution across Colombian departments using a Recharts `Treemap`. Each rectangle represents one department; its area is proportional to total sales value and its opacity scales with revenue (0–$20M COP range, using `rgba(139, 92, 246, opacity)`).

Data comes from `regionSales` in `MetricsContext`, prepared by `prepareColombiaHeatMapData(regionSales)` from `analyticsUtils`.

### Summary cards

| Card | Description |
|---|---|
| Ventas Totales | Sum of all regional sales, formatted as COP currency |
| Mejor Región | Name and value of the highest-revenue department |
| Departamentos | Count of regions with data |
| Promedio/Región | Mean revenue per department |

### Treemap visualisation

The `Treemap` component uses a custom `CustomizedContent` renderer. Department names and abbreviated revenue values (in millions, e.g. `$4.2M`) are rendered as SVG text labels inside each cell — labels only appear when the cell is wide enough (>60 px wide and >40 px tall for the name; >100 px wide for the value).

Hovering a cell shows a custom tooltip with:
- Department name
- Total sales (formatted COP)
- Order count
- Average order value

### Top-3 podium

Below the treemap, three cards highlight the top-performing departments with gold, silver, and bronze styling.

### Full regions table

A sortable table lists every department with columns: department name, sales, order count, average order value, and a percentage-of-total progress bar.

<Note>
  If `regionSales` is empty the page renders an empty-state message instead of the treemap. This can happen when no orders have associated region data.
</Note>

## Rotation ranking

**Route:** `/admin/analytics/rotacion`  
**Component:** `src/pages/admin/Analytics/RotationRanking.jsx`

Rotation ranking identifies which products move quickly (bestsellers) and which have stalled (dead stock). Products come from `ProductContext` (`apiProducts`) with a fallback to `MetricsContext` (`metricsProducts`) when the API catalogue is empty.

Two utility functions from `analyticsUtils` drive the analysis:

- `identifyBestsellers(products, orders, 10)` — returns the top 10 products ranked by `salesCount`
- `identifyDeadStock(products, orders, deadStockThreshold)` — returns products with no sales in the last N days

### Configurable dead-stock threshold

The dead-stock threshold defaults to **30 days** and is adjustable inline via a number input in the summary cards row. Changing the value immediately re-runs `identifyDeadStock` and updates the dead-stock table.

### Summary cards

| Card | Value |
|---|---|
| Bestsellers | Count of top-10 products |
| Stock Muerto | Count of products with no sales beyond the threshold |
| Total Productos | Total product catalogue size |
| Umbral Actual | Editable days input |

### Bestsellers table

The top-10 table shows rank (gold/silver/bronze styling for positions 1–3), product name and ID, units sold, total revenue, sales velocity in units per day, and category badge.

### Dead stock table

Products flagged as dead stock are listed with: product name, category, days without a sale (red), current price, and a recommendation badge generated by `identifyDeadStock`. If no products exceed the threshold the table is replaced with a success state.

<Tip>
  Lower the dead-stock threshold to 14 days for a more aggressive view of stalling inventory, or raise it to 60 days to focus only on truly abandoned products.
</Tip>

## Returns analysis

**Route:** `/admin/analytics/devoluciones`  
**Component:** `src/pages/admin/Analytics/ReturnsAnalysis.jsx`

Returns analysis cross-references `orders` and `bugReports` from `MetricsContext` to measure the impact of product quality issues on revenue.

### Summary metrics

| Card | Calculation |
|---|---|
| Tasa de Devolución | `returnedOrders.length / orders.length × 100` |
| Total Reportes | Count of all bug report entries |
| Total Pedidos | Count of all orders |
| Valor Afectado | Sum of `order.total` for returned orders |

An order is counted as returned when `order.returned === true` or `order.status === 'returned'`.

### Charts

Two Recharts panels appear side by side:

<Tabs>
  <Tab title="Reportes por tipo de tela">
    A `PieChart` breaks down bug reports by fabric type. The fabric type is extracted from the first word of `report.productName`, falling back to `report.category`. Six colours from a fixed palette (`#EF4444`, `#F59E0B`, `#10B981`, `#3B82F6`, `#8B5CF6`, `#EC4899`) cycle across slices.
  </Tab>
  <Tab title="Top vendedores con reportes">
    A horizontal bar chart lists the top 5 sellers by bug-report count. Bar width is normalised to the seller with the most reports. Each seller row also shows their return rate percentage.
  </Tab>
</Tabs>

### Quality reports table

The 10 most recent bug reports are displayed in a table with columns: ID, product name, client, description (truncated), priority badge, status badge, and date.

**Priority badges:**

| Priority | Colour |
|---|---|
| `high` | Red |
| `medium` | Yellow |
| `low` | Green |

**Status badges:**

| Status | Colour | Label |
|---|---|---|
| `open` | Blue | Abierto |
| `resolved` | Green | Resuelto |
| anything else | Gray | En revisión |

<Warning>
  Bug reports feed into this analytics view. Unresolved reports with `high` priority that appear repeatedly for the same seller are a strong signal to review that vendor in the [Vendor performance](/features/vendor-moderation#vendor-performance-dashboard) dashboard.
</Warning>

## Revenue projection

**Route:** `/admin/analytics/proyeccion`  
**Component:** `src/pages/admin/Analytics/RevenueProjection.jsx`

Revenue projection generates monthly financial forecasts from historical order and daily-sales data. Projections use an assumed 8.5 % month-on-month growth rate applied as a compound multiplier.

### Data inputs

- **Base monthly revenue** — derived from the average of the last 30 entries in `salesData` (daily), multiplied by 30. Falls back to 500,000 COP/day if no data is available.
- **Growth rate** — fixed at 8.5 % per month.

### Scenario model

For each projected month:

```
projected  = baseValue × (1 + 0.085)^i
conservative = projected × 0.85   // −15 %
optimistic   = projected × 1.20   // +20 %
```

### Projection horizon

Use the **Proyectar** dropdown to switch between 3, 6, and 12 month forecasts. The selection re-runs `generateProjection(months)` and updates all panels immediately.

### Summary cards

| Card | Description |
|---|---|
| Crecimiento Esperado | 8.5 % per month |
| Proyección N meses | Projected value for the final month |
| Escenario Conservador | Conservative value for the final month (−15 %) |
| Escenario Optimista | Optimistic value for the final month (+20 %) |

### Area chart

A Recharts `AreaChart` overlays all three scenarios with gradient fills:

- **Conservador** — amber (`#F59E0B`)
- **Proyectado** — purple (`#8B5CF6`)
- **Optimista** — green (`#10B981`)

The y-axis is in millions of COP. Tooltip values are formatted as `$X.XXM`.

### Monthly detail table

A breakdown table shows all projected months with conservative, projected, and optimistic columns plus a `±range/2` column showing the uncertainty spread for each month.

<Note>
  The 8.5 % growth rate is a demonstration default. It does not adapt to real seasonality or market changes. Use the projections as directional guidance, not financial commitments.
</Note>
