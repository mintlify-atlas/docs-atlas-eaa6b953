The checkout page (`/checkout`) is split into a two-column layout: `CheckoutForm` on the left (2/3 of the grid) and `OrderSummary` on the right (1/3, sticky at `top-24`). It requires an active cart — an empty cart triggers an immediate redirect to `/carrito`.

## Protected route behavior

```jsx
React.useEffect(() => {
    if (cartItems.length === 0) {
        navigate('/carrito');
    }
}, [cartItems, navigate]);
```

This effect runs on mount and whenever `cartItems` changes. If the cart empties (e.g. after a successful payment), the user is redirected away before they can interact with the form.

## Form fields

`Checkout.jsx` owns a single `formData` state object with these fields:

| Field | Label | Required | Validation |
|---|---|---|---|
| `fullName` | Nombre Completo | Yes | Min 3 characters |
| `email` | Correo Electrónico | Yes | RFC 5322-style regex |
| `phone` | Teléfono | Yes | Exactly 10 digits (spaces stripped) |
| `address` | Dirección | Yes | Non-empty |
| `city` | Ciudad | Yes | Non-empty |
| `state` | Departamento | Yes | Non-empty |
| `zipCode` | Código Postal | Yes | Non-empty |
| `notes` | Notas de Entrega | No | Optional; no validation |

`formData` and `setFormData` are passed as props to `CheckoutForm`, which renders the inputs and manages an `errors` state object.

## Real-time validation

`CheckoutForm` validates each field on every `onChange` event through `validateField(name, value)`:

```jsx
const handleChange = (e) => {
    const { name, value } = e.target;
    setFormData({ ...formData, [name]: value });

    const error = validateField(name, value);
    setErrors({ ...errors, [name]: error });
};
```

Error messages appear as red `<p>` tags directly below each input. Inputs with errors also receive the `border-red-500` class.

On submit, all required fields are validated in a single pass. If any produce errors, `setErrors` is called with the full error map and the form does **not** call `onSubmit`.

## Order summary

`OrderSummary` reads cart data directly from `useCart()` and re-runs the same financial calculations used on the cart page:

```jsx
const shipping = discountedSubtotal >= 100000 ? 0 : 15000;
const tax = discountedSubtotal * 0.19;
const total = discountedSubtotal + shipping + tax;
```

The summary panel shows:

1. A scrollable list of cart items (max height `16rem`) with thumbnail, name, quantity × price, and line total.
2. Subtotal, coupon discount (if any), shipping, and IVA line items.
3. Grand total.
4. A live shipping address preview that appears as soon as `formData.fullName` is non-empty.

If a coupon is applied, the `FiTag` icon, the coupon code in uppercase, and the discount amount are shown between the subtotal and shipping rows.

## Order number generation and submit flow

When the form passes validation, `CheckoutForm` calls `onSubmit()` in `Checkout.jsx`. That function:

<Steps>
  <Step title="Generate order number">
    Creates a unique order number prefixed with `DD` followed by the last 8 digits of `Date.now()`:

    ```jsx
    const orderNumber = `DD${Date.now().toString().slice(-8)}`;
    ```
  </Step>
  <Step title="Recalculate totals">
    Applies the same discount, shipping, and tax logic to produce a `total` value at the time of submission.
  </Step>
  <Step title="Persist to localStorage">
    Writes the complete `orderData` object to `localStorage` under the key `'lastOrder'`:

    ```jsx
    localStorage.setItem('lastOrder', JSON.stringify(orderData));
    ```

    The object includes `orderNumber`, `items`, `customerInfo`, `subtotal`, `discount`, `couponCode`, `shipping`, `tax`, `total`, and `date`.
  </Step>
  <Step title="Navigate to payment">
    Redirects to `/pago` where the user selects a payment method.
  </Step>
</Steps>

<Note>
  `notes` is excluded from required-field validation in `CheckoutForm`. The field still submits its value inside `formData.notes` when the order is saved.
</Note>
