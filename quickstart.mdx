---
title: "Quickstart"
description: "Get the D&D Textil store running locally in under 10 minutes."
---

This guide walks you through cloning the repo, setting up the database, starting the Java backend, and launching the React dev server.

## Prerequisites

- **Node.js** 18 or later
- **npm** 9 or later (or yarn / pnpm)
- **Java JDK** 11 or later
- **PostgreSQL** 14 or later

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Jhonmoreno000/PROYECTO-SENA-TIENDA-TEXTIL-.git
    cd "PROYECTO-SENA-TIENDA-TEXTIL-/tienda digital de telas"
    ```
  </Step>

  <Step title="Install frontend dependencies">
    <CodeGroup>
      ```bash npm
      npm install
      ```
      ```bash yarn
      yarn install
      ```
      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Set up the database">
    Create a PostgreSQL database and restore the included schema dump:

    ```bash
    createdb tienda_textil
    pg_restore -d tienda_textil "../../BASE DE DATOS/TIENDA DIGITAL TEXTIL.sql"
    ```

    <Info>
      The SQL file is a binary `pg_dump --format=custom` archive. Use `pg_restore`, not `psql`, to import it.
    </Info>

    The default connection config used by the Java backend (from `conexion/Conexion.java`):

    | Parameter | Default |
    |---|---|
    | Host | `localhost` |
    | Port | `5432` |
    | Database | `tienda_digital_textil` |
    | User | `postgres` |
    | Password | `admin` |

    Override with environment variables: `DB_URL`, `DB_USER`, `DB_PASSWORD`.
  </Step>

  <Step title="Start the Java backend">
    The backend is a plain Java HTTP server located in `backend-java/conexionPostgres/`.

    <Tabs>
      <Tab title="Windows">
        ```bash
        cd backend-java/conexionPostgres
        java -cp "bin;lib/gson-2.10.1.jar;lib/postgresql-42.7.3.jar" App
        ```
      </Tab>
      <Tab title="macOS / Linux">
        ```bash
        cd backend-java/conexionPostgres
        java -cp "bin:lib/gson-2.10.1.jar:lib/postgresql-42.7.3.jar" App
        ```
      </Tab>
    </Tabs>

    The API server starts on port **8081**. You should see:

    ```
    Servidor iniciado en puerto 8081
    ```
  </Step>

  <Step title="Start the React dev server">
    ```bash
    npm run dev
    ```

    The store opens at **http://localhost:3001** (Vite's configured port).
  </Step>

  <Step title="Log in and explore">
    Use the development credentials to test each role:

    | Role | Email | Password |
    |---|---|---|
    | Admin | `admin@ddtextil.com` | `admin123` |
    | Seller | `vendedor@ddtextil.com` | `vendedor123` |
    | Client | `cliente@ddtextil.com` | `cliente123` |

    <Warning>
      These are development credentials only. Passwords are hashed with SHA-256 and stored in PostgreSQL. Never use these credentials in production.
    </Warning>
  </Step>
</Steps>

## What's running

After completing the steps above:

| Service | URL |
|---|---|
| React frontend | http://localhost:3001 |
| Java API server | http://localhost:8081 |
| PostgreSQL | localhost:5432 |
