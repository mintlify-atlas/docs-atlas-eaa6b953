D&D Textil is a monorepo containing a React 18 frontend (`src/`), a Java HTTP backend (`backend-java/`), and a PostgreSQL database dump (`BASE DE DATOS/`).

## Full directory tree

```
tienda digital de telas/
├── src/
│   ├── components/          # Reusable UI components
│   ├── pages/               # Route-level page components
│   │   ├── admin/           # Admin dashboard pages
│   │   ├── client/          # Client dashboard pages
│   │   ├── seller/          # Seller dashboard pages
│   │   └── payment/         # Payment flow pages
│   ├── context/             # React Context providers
│   ├── hooks/               # Custom React hooks
│   ├── utils/               # Utility functions
│   ├── data/                # Static JSON data and mock data
│   ├── App.jsx              # Root component — all routes live here
│   ├── main.jsx             # Vite entry point
│   └── index.css            # Global styles (Tailwind base)
├── backend-java/
│   └── conexionPostgres/
│       ├── src/
│       │   ├── App.java             # Entry point — starts the HTTP server
│       │   ├── api/                 # HTTP request handlers (ApiServer, AuthHandler, …)
│       │   ├── conexion/            # PostgreSQL connection singleton
│       │   ├── dao/                 # Data access objects
│       │   └── models/              # Plain Java model classes
│       ├── lib/
│       │   ├── gson-2.10.1.jar      # JSON serialization
│       │   └── postgresql-42.7.3.jar # JDBC driver
│       └── bin/                     # Compiled .class files
├── public/                  # Vite static assets
├── BASE DE DATOS/
│   └── TIENDA DIGITAL TEXTIL.sql   # PostgreSQL custom-format dump
├── package.json
├── vite.config.js
└── tailwind.config.js
```

### `context/`

Six React Context providers manage all global state. They are composed inside `AppProviders.jsx`. See the [State management reference](/reference/state-management) for full API details.

| File | Hook | Purpose |
|---|---|---|
| `NotificationContext.jsx` | `useNotification` | Toast notifications |
| `AuthContext.jsx` | `useAuth` | Authentication and roles |
| `MetricsContext.jsx` | `useMetrics` | Admin data, orders, products, coupons |
| `CartContext.jsx` | `useCart` | Shopping cart |
| `ProductContext.jsx` | `useProducts` | Public product catalog |
| `UIContext.jsx` | `useUI` | Carousel slides, home section visibility |

### `hooks/`

| File | Purpose |
|---|---|
| `useLocalStorage.js` | `useState`-compatible hook that syncs to `localStorage` |
| `useDarkMode.js` | Persists dark mode preference and toggles the `dark` class on `<html>` |

### `utils/`

| File | Purpose |
|---|---|
| `formatters.js` | Number and currency formatting helpers |

### `data/`

| File | Purpose |
|---|---|
| `products.json` | Static product catalogue used as fallback when the backend is unreachable |
| `mockData.js` | Mock users, orders, sales data, bug reports, inventory batches, coupons, support tickets, and regional sales — used as initial state in `MetricsContext` |

## Backend — `backend-java/conexionPostgres/`

A plain Java HTTP server using `com.sun.net.httpserver.HttpServer`. No framework dependency — only Gson for JSON and the PostgreSQL JDBC driver.

| Package / file | Purpose |
|---|---|
| `App.java` | Entry point: validates DB connection, starts API on port `8081` |
| `conexion/Conexion.java` | Singleton `Connection` with automatic reconnection logic |
| `api/ApiServer.java` | Registers all HTTP contexts and contains inline `HttpHandler` classes for products and static files |
| `api/AuthHandler.java` | Handles `POST /api/login` and `POST /api/register` |
| `api/UsersHandler.java` | Handles `GET /api/users` |
| `api/OrdersHandler.java` | Handles `GET /api/orders` and `PUT /api/orders/:id/status` |
| `api/CouponsHandler.java` | Handles CRUD on `/api/coupons` |
| `api/ConfigHandler.java` | Key–value config store on `/api/config` |
| `api/SupportHandler.java` | Handles `/api/support/tickets` and `/api/support/bugs` |
| `dao/` | One DAO class per entity with direct JDBC queries |
| `models/` | Plain Java beans: `User`, `Product`, `Order`, `Coupon`, `SupportTicket`, `BugReport`, `ConfigItem` |

## Starting the project

```bash
# 1. Install frontend dependencies
npm install

# 2. Start the Java backend (must run first)
cd backend-java/conexionPostgres
java -cp "bin;lib/gson-2.10.1.jar;lib/postgresql-42.7.3.jar" App

# 3. Start the React dev server (new terminal)
npm run dev
```

<Note>
  The frontend falls back to `src/data/products.json` and mock data from `src/data/mockData.js` when the backend is unavailable. The app remains functional without the Java server for UI development.
</Note>
