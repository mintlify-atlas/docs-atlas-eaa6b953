---
title: 'Routing'
description: 'Complete route reference for D&D Textil — every path, its component, and the required access level.'
---

All routes are defined in `src/App.jsx` using React Router v6 `<Routes>` inside `<BrowserRouter>`. Route transitions are animated with Framer Motion `<AnimatePresence mode="wait">`.

Access control is enforced by the `ProtectedRoute` component (`src/components/ProtectedRoute.jsx`), which checks `isAuthenticated` and optionally validates `user.role` against a required role or list of roles from `AuthContext`.

## Public routes

| Path | Component | Description |
|---|---|---|
| `/` | `pages/Home` | Home page with hero, carousel, featured products, and benefits |
| `/catalogo` | `pages/Catalog` | Product catalogue with filters and sorting |
| `/producto/:id` | `pages/ProductDetail` | Individual product page with image gallery and add-to-cart |
| `/carrito` | `pages/Cart` | Shopping cart with quantity controls |
| `/nosotros` | `pages/About` | About D&D Textil page |
| `/contacto` | `pages/Contact` | Contact form page |
| `/login` | `pages/Login` | Login form |
| `/registro` | `pages/Register` | New account registration |
| `/reportar-fallo` | `pages/BugReport` | Public bug / issue report form |
| `/loader-demo` | `pages/LoaderDemo` | Development demo of the `TruckLoader` animation |

## Role normalization

Database roles are stored in Spanish. `AuthContext` normalizes them to English on login:

```jsx
if (normalizedRole === 'administrador' || normalizedRole === 'admin') normalizedRole = 'admin';
else if (normalizedRole === 'vendedor') normalizedRole = 'seller';
else if (normalizedRole === 'cliente') normalizedRole = 'client';
```

`ProtectedRoute` then compares against the normalized value:

```jsx
// Single role
<ProtectedRoute roles="client">
  <ClientDashboard />
</ProtectedRoute>

// Multiple roles
<ProtectedRoute roles={['seller', 'admin']}>
  <SellerProducts />
</ProtectedRoute>
```

<Note>
  The `roles` prop is optional on `ProtectedRoute`. When omitted, only authentication is checked — any logged-in user can access the route regardless of role.
</Note>
